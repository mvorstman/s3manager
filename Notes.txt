python3 - <<'PY'
import os
import random
from pathlib import Path

root = Path.home() / "s3manager-test" / "temp-8155-uploadset"
root.mkdir(parents=True, exist_ok=True)

random.seed(42)

file_count = 2000
max_depth = 4
folder_span = 6

for i in range(file_count):
    depth = random.randint(0, max_depth)
    parts = [f"folder{random.randint(1, folder_span)}" for _ in range(depth)]
    d = root.joinpath(*parts)
    d.mkdir(parents=True, exist_ok=True)

    filename = f"file_{i:06d}.txt"
    size = random.choice([1024, 2048, 4096, 8192])

    with open(d / filename, "wb") as f:
        f.write(os.urandom(size))

print(f"Created {file_count} files under {root}")
PY

===== SAFETY & USAGE NOTES =====
- Never run delete on an empty prefix (risk of deleting entire bucket)
- Always validate with list before delete
- Prefer implementing and using a --dry-run flag for delete operations
- Use dedicated prefixes per test (avoid overlap)
- Download to a clean folder to validate results
- Test dataset is stable and reproducible
- Workload is I/O bound
- Optimal worker range currently ~40-60


===== GIT WORKFLOW =====

===== WHEN TO COMMIT =====
- After completing a logical feature (e.g. worker pool, retry, progress)
- After fixing a bug
- Before starting a risky change
- At the end of a working session


===== COMMIT MESSAGE STYLE =====
- Keep it short and clear
- Describe what changed, not how

Examples:
- "Add parallel upload-folder with worker pool"
- "Add throughput reporting to upload summary"
- "Implement retry with exponential backoff"


===== GOOD PRACTICES =====
- Commit small, meaningful changes (not everything at once)
- Avoid committing broken code when possible
- Use tags only for stable milestones
- Push regularly to avoid losing work
- Keep DEVLOG.txt aligned with major commits
