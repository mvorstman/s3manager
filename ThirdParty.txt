================================================================================
  PROJECT OVERVIEW
================================================================================

S3Manager is a Go project evolving from a CLI tool into a server-based web
application with a browser UI, similar in spirit to S3 Explorer. It manages
S3-compatible object storage.
The PowerShell companion (S3-Manager-v2.5.ps1) was retired on 2026-04-12.

  Go CLI (main.go + internal/)
     - Flag-based CLI; connects to any S3-compatible endpoint
     - Credentials via flags or env vars (S3_ENDPOINT, AWS_ACCESS_KEY_ID,
       AWS_SECRET_ACCESS_KEY, AWS_REGION)
     - Actions: list, upload, upload-folder, download, download-prefix, head, delete
     - Concurrency: configurable worker pool for upload-folder, download-prefix, delete
     - Retry: configurable max-attempts + backoff via AWS SDK v2

  Architecture (Go):
    internal/s3/       — low-level S3 primitives (client, uploader, downloader,
                         lister, head). Each file is a focused, testable unit.
    internal/engine/   — orchestration: worker pools, pagination loops, batch
                         delete, folder walk, result aggregation.
    main.go            — CLI entry point; flag parsing, env fallback, dispatch.

  Backends (known targets):
    - StorageGRID  — primary, public endpoint, valid CA cert
    - ActiveScale  — secondary
    - MinIO        — local fallback / development

2026-04-12 — Credential management design decision (discussed on iPad):

  Three options evaluated:
    Option 1 - Env vars: simple, current approach, no multi-user support
    Option 2 - SQLite + AES-256-GCM: per-user encrypted credentials,
               bcrypt passwords, one master key in env var
    Option 3 - Envelope encryption: per-user keys wrapped by master key
               (KMS-style), most secure, high complexity

  Decisions:
    - Start with Option 1
    - Migrate to Option 2 when multi-user is needed
    - Option 3 is a long-term possibility
    - Build in-house using Go standard library only (no Vault etc.)
    - Migration impact is low: only client.go, credentials.go, main.go
    - Isolate behind a CredentialProvider interface, defined early
