===== S3MANAGER PROJECT =====

S3Manager is a Go-based CLI tool for managing S3-compatible object storage systems such as:
- NetApp StorageGRID
- Quantum ActiveScale
- MinIO

This project is both:
- a production-grade tooling effort
- a structured learning path in Go, system design, and S3 internals

---

===== GOALS =====

1. Become an S3 expert
   - Deep understanding of S3 APIs and behavior
   - Understand performance characteristics at scale

2. Build production-grade tooling
   - Reliable
   - Predictable performance
   - Safe for destructive operations

3. Learn system design
   - Concurrency patterns
   - Streaming pipelines
   - Backpressure handling
   - Observability

---

===== ARCHITECTURE =====

- main -> CLI / argument parsing / user-facing output
- engine -> orchestration, worker pool logic, aggregation
- internal/s3 -> low-level S3 operations

Design rules:
- Workers do NOT print
- Workers do NOT aggregate
- Engine aggregates results
- S3 layer only performs low-level S3 work
- Strict separation between discovery, execution, and presentation

---

===== CURRENT STATUS =====

- Cleanup phase completed
- CLI contract standardized (single --prefix flag)
- delete pipeline implemented with batching and worker pool
- configurable batch size supported (--batch-size)
- lightweight instrumentation added to delete pipeline:
  - total batches
  - average batch size
  - delete latency (avg/min/max)
  - enqueue wait (avg/max)
  - deletes/sec
- code builds cleanly (go build ./...)

Key findings from latest tests:
- batching is efficient (~950 avg size)
- enqueue wait is negligible (no backpressure)
- delete API latency dominates runtime
- bottleneck is backend/API path, not batching stage
